Navigating the File System (code+meaning+example)

Code

Meaning

Example

$

The command line allows a user to navigate the filesystem and run built-in programs or custom scripts. In Unix, the command line interface is called Bash, and the shell prompt is the $.

$

pwd

displays the file path from the root directory to the current working directory

$ pwd
/users/Maria/Downloads

mkdir

Makes a new directory according to it's argument. If a file path is given, the new directory will be placed within that directory

$ mkdir new-directory
ls
old-directory new-directory

ls

Is used to list the contents of a directory. If no arguments are given, it will list the contents of the current working directory

$ ls Desktop
resume.pdf
photo.png

cd

Is used to move throughout the filesystem of a computer. It accepts a variety of arguments:

-full file paths
-names of children of the current directory
-.. the parent of the current directory

$ cd some-directory
$ cd ..

touch

creates a new file in the current working directory with the name provided in the argument

$ touch grocery-list

Viewing and Changing the File System (code+meaning+example)

Code

Meaning

Example

sort

Takes the standard input and order it alphabetically for the standard output (it doesn't change the file)

sed

Similar to find and replace. s = the substitute command, it tells sed that you want to perform a substitution operation. It's the first character in the expression and is always follower by a delimiter (commonly a slash /). Changes won't be made to the file. Use -i option makes changes

$ sed "s/find/replace/g" file.txt

cp

Is used to copy files or directories. The basic argument structore is: cp source destination, where the source is the file/directory to copy to the destination file/directory

$ cp file1 file1_copy
$ cp file1 file2 destination_folder

.bak is commonly used to notate a file as a backup of a file with the same name

mv

Is used to move a file into a directory. Use mv with the source file as the first argument and the destination folder as the second argument

$ mv index.html website/

rm

Is used to delete files and directories. The -r flag deletes a directory and all of its files and directories (rm -r).

$ rm -r bad_selfies

ls <options>

Is used to list the contents of a directory. It can be used combined with the following command options:

$ ls -a
$ ls -l
$ ls -t

Redirecting Input and Output

Code

Meaning

Example

>>

The >> is used to redirect the standard output of the command on the and append it (add) it to the end of the file on the right.

echo "Hello World!" >> greeting.txt

|

Is used to pipe, or transfer, the standard output from the command on it's into the standard input of the command on it's right

echo "Hello World" | wc -w (uses word count command with word option to output the words)

>

Is used to redirect output by taking the output from the command on the and passing as input to the file on the right.

echo "Hello" > hello.txt

cat

displays the contents of one or more files to the terminal.

$ cat poem.txt
$ cat poem.txt kitties.txt

grep

is used to search files for lines that match a pattern and returns the results. Various options can be specified along with the grep command to specify the search. An option of -i can be used to search for lines that match a pattern, case insensitive, and returns the result.

grep 'sonny' names.txt

grep -R

The shell command grep has a -R option (grep -R) that searches all files in a directory, including its subdirectories, and outputs filenames and lines containing matched results.

grep options

  • -c
  • -h
  • -i
  • -l
  • -n
  • -v
  • -w

  • -c prints only a count of the lines that match a pattern
  • -h display the matched lines, but do not display the filenames
  • -i ignores, case for matching
  • -l displays list of filenames only
  • -n display the matched lines and their line numbers
  • -v prints out all the lines that do not match the pattern
  • -w match the whole word

uniq

Filters ot adjacent duplicate lines in a file (best if used with sort to eliminate duplicate entries)

>
>>

On a command line, redirection is the process of using the input/output of a file or command to use it as an input for another file. It is similar but different from pipes, as it allows reading/writing from files instead of only commands.

ls > directories_list.txt
ls >> directories_list.txt

Configuring the Environment

Code

Meaning

Example

env

For Unix-based systems like Mac OS and Linux (not Windows), the shell command env returns a list of environment variables for the current user.

alias

is used to assign commonly used commands to shortcuts (or aliases). The assigned commonly used command should be wrapped in double quotes.

source

All the commands in ~/.bash_profile are executed with the shell command source ~/.bash_profile. So when changes are made to ~/.bash_profile, run this command to activate the changes in the current session.

history

Is used to get a history of commands (also known as “events”) that were executed in the current session. The command also allows us to perform operations on this list of commands that have been executed, such as selecting or manipulating a command in the history.

export

Makes a given variable available to all child sessions initiated from the current session.

export USER="Jane Doe"

HOME

is an environment variable present in command line environments. It is used to get the path to the current user’s home directory. This makes it easy for programs to access the home directory when needed.

echo $HOME
